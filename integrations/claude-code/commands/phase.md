# Current Phase Status

Show the current workflow phase, what it means, and what to do next.

## What to Display:

1. **Current Phase**: [from PROJECT-STATE.md or ask user]
2. **Phase Name**: e.g., "Phase 1.2: RESEARCH"
3. **Purpose**: What this phase accomplishes
4. **Current Step**: What should happen now
5. **Stop & Wait?**: Is this a checkpoint?
6. **Next Action**: Specific instruction for user or AI

## Phase Quick Reference:

- **Phase 0**: Triage - Classify request
- **Phase 1.1**: Discovery - Ask clarifying questions
- **Phase 1.2**: Research - Check existing code
- **Phase 1.3**: Brainstorm - Present 3 approaches
- **Phase 1.4**: Evaluation - Recommend approach
- **Phase 1.5**: Premortem - Identify risks
- **Phase 1.6**: Specification - Create implementation plan
- **Phase 1.7**: Approval - Get user GO signal
- **Phase 2.1-2.4**: Coding - Implement step-by-step
- **Phase 3.1-3.3**: Testing - Verify and fix
- **Phase 4.1-4.3**: Review & SEAL Learning

## Usage:

User types: `/phase` or `/phase 1.2`
Result: Clear status update and guidance on next steps
